# 个人主页本地内容管理后台 (Local CMS)

这是一个专门为了维护个人主页内容而编写的轻量级本地管理后台工具。
通过它，你可以直观地增删改查主页中的“项目”和“世界”板块内容，以及直接上传本地图片。

## ⚠️ 重要说明（不会影响线上部署）
- 这个 `admin` 文件夹**仅仅作为本地修改内容的工具存在**。
- GitHub Pages 在部署时只会读取根目录下的 `index.html` 以及 `css/`、`js/` 等静态文件。
- GitHub Pages 是一个纯静态托管环境，**不会运行**这里的 Node.js 服务。所以即使你把它 push 到了 GitHub 上，也**完全不会影响你的主页正常显示和部署**，只会作为源码保存下来。

## 🚀 如何在本地使用它？

### 1. 环境准备
确保你的电脑上安装了 [Node.js](https://nodejs.org/)。

### 2. 安装依赖
打开终端，进入 `admin` 目录，安装运行所需的依赖库：
```bash
cd admin
npm install
```

### 3. 启动后台服务
运行以下命令启动服务：
```bash
node server.js
```
终端会提示你服务已启动，此时请在浏览器中打开：
**http://localhost:3000**

### 4. 使用方式
1. **修改内容**：在后台界面中修改“项目”或“世界”的文字、图片。
2. **上传图片**：点击“上传图片”，图片会自动重命名并保存到项目根目录的 `images/projects/` 或 `images/travels/` 文件夹下。
3. **保存数据**：点击右上角的“**保存所有修改到 data.js**”按钮，数据会自动写入到根目录的 `js/data.js` 文件中。
4. **发布上线**：确认修改无误后，像平时一样在项目根目录执行 `git add .`、`git commit` 和 `git push`。GitHub Pages 会自动更新你的主页！

## 🛠 技术栈
- 后端：Node.js + Express + Multer（用于图片上传）
- 前端：Vue 3 (CDN) + Tailwind CSS (CDN)
- 数据库：无（直接读写本地的 `../js/data.js` 文件）
